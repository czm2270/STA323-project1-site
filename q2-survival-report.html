Q2 Survival Analysis Report

Project 1 | PySpark AFT Survival Regression

1. Task Objective

This task aims to model time-to-event outcomes using survival analysis and reproduce the workflow in a single notebook with clear comments.

2. Method

We use AFTSurvivalRegression in PySpark. The pipeline includes: data preparation, feature vector construction, train/test split, model fitting, and prediction with quantiles.

3. Data and Features

  • time_months: observed duration until event or censoring
  • event: 1 means event happened, 0 means censored
  • Feature examples: usage_intensity, price_sensitivity

4. Key Results

  • Coefficients: [-4.598446, -6.360452]
  • Intercept: 9.054665
  • Scale: 0.090641

Test-set predictions and quantiles were generated successfully, confirming the pipeline is executable and reproducible.

5. Interpretation

In this sample setting, higher usage intensity and higher price sensitivity are associated with shorter predicted survival time under the fitted AFT model. These are model-based associations and should not be interpreted as strict causality.

6. Limitations

The current sample size is small and mainly for reproducibility. For a production-grade analysis, larger real datasets and additional model comparisons (e.g., Kaplan-Meier, Cox) are recommended.
