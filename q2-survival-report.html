Q2 Survival Analysis Report

Project 1 | PySpark AFT Survival Regression

1. Task Objective

The goal is to model time-to-event outcomes and reproduce the survival analysis workflow in one notebook with clear comments.

2. Data and Problem Setup

I used a reproducible sample dataset with these fields: time_months (observed duration), event (1=event happened, 0=censored), and two numeric features usage_intensity, price_sensitivity.

3. Method

I applied AFTSurvivalRegression in PySpark. The pipeline is: data preparation → feature vector construction → train/test split → model fitting → prediction and quantile output.

4. Key Results

  • Coefficients: [-4.598446, -6.360452]
  • Intercept: 9.054665
  • Scale: 0.090641

The test set prediction table and quantiles were generated successfully, which means the model pipeline ran correctly end-to-end.

5. Interpretation

In this run, both feature coefficients are negative. Under this AFT setting, larger values of these features are associated with shorter predicted survival time. This is an association pattern from the fitted model, not strict causality.

6. Practical Notes

I encountered optimization warnings (e.g., line-search step adjustment), but the training still converged and produced stable outputs. So I kept the results and verified them in the notebook instead of discarding the run.

7. Limitations and Next Steps

The current sample is small and mainly for reproducibility. In future work, I would test larger real data and compare additional methods such as Kaplan-Meier and Cox models.
